Hugo에 Hextra 테마 적용하는 가장 쉬운 방법

June 26, 2025

정적 사이트 생성기(Static Site Generator, SSG)인 휴고(Hugo)는 빠른 속도와 강력한 기능으로 많은 사랑을 받고 있습니다. 특히 마크다운으로 콘텐츠를 작성하고, 다양한 테마를 적용하여 손쉽게 자신만의 블로그나 웹사이트를 만들 수 있다는 점이 큰 매력입니다.

수많은 휴고 테마 중에서도 Hextra는 깔끔한 디자인과 뛰어난 기능성으로 최근 많은 주목을 받고 있습니다. 오늘은 휴고에 Hextra 테마를 적용하는 가장 간단하고 확실한 방법을 알아보겠습니다.

Hextra 테마, 왜 좋은 선택일까요?#

Hextra 테마는 다음과 같은 장점을 가지고 있어 많은 개발자들에게 좋은 평가를 받고 있습니다.

  • 모던하고 깔끔한 디자인: 콘텐츠에 집중할 수 있는 미니멀한 디자인을 제공합니다.
  • 다크 모드 지원: 사용자의 시스템 설정에 따라 자동으로 다크 모드를 지원하여 눈의 피로를 덜어줍니다.
  • 강력한 검색 기능: 내장된 FlexSearch를 통해 빠르고 정확한 전체 텍스트 검색 기능을 제공합니다.
  • 다양한 컴포넌트: 경고, 탭, 카드 등 다양한 UI 컴포넌트를 기본적으로 지원하여 콘텐츠를 다채롭게 꾸밀 수 있습니다.
  • 쉬운 설치와 설정: 휴고 모듈(Hugo Modules)을 지원하여 몇 가지 명령어만으로 간단하게 설치하고 업데이트할 수 있습니다.

Hextra 테마 적용 따라하기#

이제 본격적으로 여러분의 휴고 사이트에 Hextra 테마를 적용해 보겠습니다. 터미널(Terminal) 또는 명령 프롬프트(Command Prompt) 창을 열고 차근차근 따라오세요.

사전 준비:

  • Hugo가 설치되어 있어야 합니다.
  • Git이 설치되어 있어야 합니다.

1단계: 새로운 휴고 사이트 생성#

먼저, Hextra 테마를 적용할 새로운 휴고 사이트를 생성합니다. 이미 기존에 운영하던 사이트가 있다면 이 단계는 건너뛰어도 좋습니다.

hugo new site my-hextra-site
cd my-hextra-site

위 명령어는 my-hextra-site라는 새로운 폴더를 만들고 해당 폴더로 이동하는 명령어입니다.

2단계: Git 저장소 초기화#

휴고 모듈을 사용하기 위해서는 해당 프로젝트 폴더가 Git 저장소로 관리되어야 합니다. 다음 명령어를 실행하여 Git 저장소를 초기화합니다.

git init

3단계: 휴고 모듈 초기화 및 테마 추가#

이제 가장 중요한 단계입니다. 휴고 모듈을 초기화하고 Hextra 테마를 프로젝트에 추가합니다.

기존에는 config.toml 또는 hugo.toml 파일을 직접 수정하고 테마 파일을 다운로드하는 복잡한 과정이 필요했습니다. 하지만 휴고 모듈을 사용하면 이 과정이 매우 간단해집니다.

다음 명령어를 터미널에 입력하세요.

hugo mod init github.com/<your_username>/<your_repo>
hugo mod get -u github.com/imfing/hextra
  • hugo mod init ...: 내 프로젝트를 휴고 모듈로 사용하겠다고 선언하는 과정입니다. <your_username>과 <your_repo> 부분은 여러분의 GitHub 사용자 이름과 저장소 이름으로 변경해주세요. (예: github.com/my-id/my-blog)
  • hugo mod get ...: Hextra 테마를 원격 저장소에서 다운로드하고 필요한 설정을 자동으로 추가하는 명령어입니다. -u 옵션은 최신 버전의 테마를 가져오도록 합니다.

4단계: hugo.toml 파일 설정#

테마를 적용하기 위해 휴고 설정 파일을 수정해야 합니다. 프로젝트의 최상위 경로에 있는 hugo.toml (또는 config.toml) 파일을 열고 theme 설정값을 추가합니다.

baseURL = 'https://example.org/'
languageCode = 'ko-kr'
title = 'My New Hugo Site'
theme = 'github.com/imfing/hextra' # 이 부분을 추가하거나 수정합니다.

theme 값으로 방금 전 hugo mod get 명령어로 내려받은 테마의 경로(github.com/imfing/hextra)를 정확하게 입력하는 것이 중요합니다.

5단계: 로컬 서버 실행 및 확인#

모든 설정이 완료되었습니다. 이제 로컬 서버를 실행하여 Hextra 테마가 잘 적용되었는지 확인해 봅시다.

hugo server

[필요한 이미지: hugo server 명령 실행 후 터미널에 나타나는 결과 화면]

위 명령어를 실행하면 터미널에 Web Server is available at http://localhost:1313/ 와 같은 메시지가 나타납니다. 웹 브라우저를 열고 해당 주소(http://localhost:1313)로 접속하면 Hextra 테마가 적용된 여러분의 사이트를 확인할 수 있습니다.

[필요한 이미지: Hextra 기본 테마가 적용된 로컬 웹사이트 초기 화면 스크린샷]

3. 문제 해결 (Troubleshooting)#

만약 테마가 정상적으로 적용되지 않는다면 다음 사항을 확인해 보세요.

  • hugo.toml 파일의 theme 설정: theme 값에 오타가 없는지, 정확한 테마 경로가 입력되었는지 다시 한번 확인합니다.
  • 휴고 버전: Hextra 테마는 최신 버전의 휴고에서 가장 잘 동작합니다. hugo version 명령어로 현재 버전을 확인하고, 필요하다면 최신 버전으로 업데이트하는 것을 권장합니다.
  • 기존 콘텐츠와의 충돌: 기존에 다른 테마를 사용하고 있었다면, 이전 테마의 설정이나 콘텐츠의 프론트 매터(Front Matter) 부분이 충돌을 일으킬 수 있습니다. Hextra 테마의 공식 문서를 참고하여 콘텐츠 구조를 확인해 보세요.

참조:

  • Hextra 공식 문서: https://imfing.github.io/hextra/
  • Hugo Modules 공식 문서: https://gohugo.io/hugo-modules/

Hextra 테마는 휴고 모듈을 통해 매우 간단하게 설치하고 관리할 수 있습니다. 오늘 알려드린 방법으로 여러분의 휴고 사이트를 더욱 세련되고 기능적으로 업그레이드해 보시길 바랍니다.
